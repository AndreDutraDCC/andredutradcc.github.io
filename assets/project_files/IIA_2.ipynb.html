Parte 1: k-Nearest Neighbors

O k-Nearest Neighbors, ou k-NN, um algoritmo de classificao utilizado para prever uma determinada caracterstica discreta de uma instncia de dados partir da generalizao de um conjunto de treino semelhante fornecido. No caso da base Iris, o objetivo do k-NN prever, a partir das dimenses das ptalas e das spalas, a espcie de uma instncia qualquer, partir das informaes da base usadas para o treino do algoritmo.

O k-Nearest Neighbors , desse modo, um tipo de algoritmo baseado em aprendizado supervisionado. O resultado esperado do algoritmo, a espcie da planta, j conhecido nos dados, e as classificaes do conjunto de treino so usadas para aperfeioar o resultado da classificao da instncia.

O princpio do k-Nearest Neighbors bem simples. Os dados contnuos da base, dos quais a classe ser predita, podem ser vistos como pontos no espao. No caso da base Iris, as informaes das ptalas e das spalas formariam um conjunto de pontos em um espao tetradimensional. Desse modo, dada uma base de treino, uma instncia e um determinado valor de k, o k-Nearest Neighbors seleciona os k pontos da base mais prximos da instncia, a partir de alguma mtrica de distncia, e atribui instncia analisada a classe que ocorre mais vezes entre os k. No caso da base Iris, ele escolheria, dentre as trs, a espcie a que pertencem a maioria dos elementos entre os k escolhidos e atribuiria flor que se quer classificar.

O cdigo a seguir uma implementao do k-NN. Ele utiliza como mtrica de distncia a distncia euclidiana, definida na funo euclidean. Ela simplesmente toma duas linhas da tabela, filtra apenas as colunas de dados, sobre as quais a distncia ser calculada, e calcula a distncia entre os pontos fazendo a norma vetorial da diferena dos pontos.

O k-NN em si est implementado na funo kNN_classify, e recebe como entrada o nmero de vizinhos k, que deve ser um inteiro, a base de treino data, que deve ser uma tabela na qual uma das colunas a categoria de classificao e as demais os demais dados, e a instncia instance a ser testada, que deve seguir o mesmo modelo da tabela data (exceto pela coluna de categoria). Opcionalmente, o nome da coluna na qual a categoria a ser classificada pode ser fornecida no parmetro category_col. Por padro, se ela no for fornecida, o algoritmo seleciona a ltima coluna da tabela data

def euclidean(x, y, data_cols):
    return np.linalg.norm(x[data_cols]-y[data_cols])

def kNN_classify(k, data, instance, category_col = None):
    if category_col is None:
        category_col = data.columns[-1]
    
    data_cols = [col for col in data.columns if col != category_col]
    
    k_nearest = data.head(k).copy()
    k_nearest['dist'] = [euclidean(row, instance, data_cols) for _,row in k_nearest.iterrows()]
    
    idx_max = k_nearest['dist'].idxmax()
    max_distance = k_nearest['dist'][idx_max]
    
    for _, row in data.iterrows():
        distance = euclidean(row, instance, data_cols)

        if distance >= max_distance:
            continue
          
        row['dist'] = distance
        k_nearest.loc[idx_max] = row

        idx_max = k_nearest['dist'].idxmax()
        max_distance = k_nearest['dist'][idx_max]
    
    return k_nearest[category_col].mode()[0]

O cdigo em si funciona de forma muito simples e intuitiva. Ele primeiro escolhe k linhas quaisquer da base para ser os k-vizinhos da instncia, e para cada uma calcula a distncia euclidiana dela instncia dada. Dentre elas, selecionado o ndice da linha mais distante entre as k, em conjunto com o valor de sua distncia.

O algoritmo ento itera por cada linha da base, calculando sua distncia euclidiana da instncia e comparando com a distncia do vizinho mais distante entre os k. Se ela estiver mais prxima da instncia que ele, a linha substitui o vizinho mais distante no conjunto dos k vizinhos e o novo vizinho mais distante recalculado. Aps passar por todas as linhas, o algoritmo retorna a moda da coluna de categorias dos vizinhos, ou seja, a categoria que ocorre com mais frequncia entre os k vizinhos mais prximos.

Observe que o algoritmo sempre encontra os k-vizinhos mais prximos. Isso ocorre por dois motivos. Primeiro, um ponto entra no conjunto dos k se, e somente se ele estiver mais prximo que o mais distante entre eles, ento um ponto mais distante testado nunca adicionado no conjunto e um ponto mais prximo testado nunca ignorado do conjunto. Segundo, todos os pontos da base so testados. Assim, apenas pontos mais prximos so adicionados entre os k, resultando nos k-vizinhos mais prximos.

Um ponto muito importante do k-NN o fato de seu funcionamento depender da escolha do k adequado para a base utilizada. Assim, tendo implementado o algoritmo, precisamos testar sua eficcia, levando em considerao principalmente a escolha de diferentes valores de k. Para isso, usaremos a diviso do banco mencionada anteriormente. 120 linhas sero utilizadas como treino e as demais 30 como teste.

Nela, as linhas e colunas contm as classes da classificao, sendo as linhas o valor real e as colunas o valor predito pelo algoritmo. Assim, para cada instncia do conjunto de teste, o k-NN executado e seu resultado, em conjunto com a classe real j existente na base, determinam a que posio da matriz de confuso a instncia pertencer. def kNN_test_data(k, train_data, test_data, category_col = None):
    if category_col is None:
        category_col = train_data.columns[-1]
    
    categories = sorted(train_data[category_col].unique())

    confusion = pd.DataFrame(0, index = categories, columns = categories)
    
    for _, row in test_data.iterrows():
        predicted = kNN_classify(k, train_data, row, category_col)
        real = row[category_col]
        confusion.loc[real,predicted] += 1

    confusion.index = pd.MultiIndex.from_product([['Real'],categories])
    confusion.columns = pd.MultiIndex.from_product([['Previsto'],categories], names = ['','k = {}'.format(k)])

    return confusion

Por essa razo, para cada matriz, foram calculados os valores de acurcia, preciso, revocao e F1:

  • A acurcia uma mtrica geral da matriz, e mede a porcentagem de instncias identificadas corretamente, ou seja, do total, quantas se encontram na diagonal da matriz.

  • A preciso uma mtrica aplicada a cada classe, e mede a porcentagem de instncias previstas corretamente dentre todas as previses da classe. Assim, uma preciso baixa indica que muitas instncias fora da classe real foram preditas como pertencentes quela classe, ou seja, um alto nmero de falsos positivos.

  • A revocao uma mtrica aplicada a cada classe, e mede a porcentagem de instncias previstas corretamente dentre todas as instncias reais da classe. Assim, uma revocao baixa indica que muitas instncias da classe real no foram preditas corretamente, ou seja, um alto nmero de falsos negativos.

  • O F1 uma forma de observar a preciso e a revocao de cada classe simultaneamente, fazendo uma mdia harmnica entre as duas. Uma das principais caractersticas da mdia harmnica que se um dos elementos for nulo, a mdia nula tambm. Assim, para que uma classe tenha um bom F1 ela precisa ter tanto uma preciso quanto uma revocao boa.

O clculo das quatro mtricas feito nas seguintes funes:

In[7]:
def accuracy(confusion):
    matrix = confusion.to_numpy()
    acc = matrix.trace()/matrix.sum()
    return pd.DataFrame(acc, index=['Acurcia'], columns = confusion.columns.droplevel(0))

def precision(confusion):
    diags = pd.DataFrame([np.diag(confusion)], index=['Preciso'], columns=confusion.columns.droplevel(0))
    predicts = confusion.sum().to_frame().transpose()
    
    predicts.columns = predicts.columns.droplevel(0)
    predicts.index = ['Preciso']
    return diags/predicts

def recall(confusion):
    diags = pd.DataFrame([np.diag(confusion)], index=['Revocao'], columns=confusion.columns.droplevel(0))
    reals = confusion.sum(axis=1).to_frame().transpose()
    
    reals.columns = reals.columns.droplevel(0)
    reals.index = ['Revocao']
    return diags/reals

def f1(confusion):
    p = precision(confusion)
    p.index = [0]
    r = recall(confusion)
    r.index = [0]

    f_1 = 2*p*r/(p+r)
    f_1.index = ['F1']
    return f_1

Parte 2: k-Means

Alm de realizar classificaes, algoritmos de aprendizado podem tambm obter novas informaes ainda no conhecidas sobre os dados. Este o princpio do aprendizado no-supervisionado, que, ao contrrio do aprendizado supervisionado, no trabalha utilizando resultados esperados para o conjunto de treino, j que estes resultados geralmente ainda no so conhecidos. Assim, algoritmos de aprendizado no-supervisionado geralmente analisam todo o conjunto de dados em busca de associaes entre as instncias.

Dentre estes algoritmos esto os algoritmos de agrupamento, que buscam agrupar os dados em conglomerados que compartilhem entre si informaes semelhantes. No caso do dataset Iris, que estamos analisando, uma aplicao imediata do mtodo seria tentar utiliz-lo para classificar as flores em grupos que correspondam s espcies, por exemplo.

O algoritmo de agrupamento que implementaremos ser o k-Means. Seu funcionamento tambm relativamente simples, e seu objetivo , dada uma base de dados e um nmero k de grupos, divid-la em k grupos de modo que pontos em um mesmo grupo estejam mais prximos entre si em relao aos demais.

Para isso, o k-Means comea selecionando k pontos aleatrios da base para cerem os centroides de cada grupo, e associando cada ponto da base ao grupo do centroide mais prximo a ele. O algoritmo ento entra em um loop, no qual ele recalcula os centroides, fazendo a mdia dos pontos de cada grupo (vide o nome, k-Means), e reagrupando os pontos de acordo com o centroide mais prximo. Quando todos os pontos continuarem no mesmo grupo aps uma iterao do loop, os grupos convergiram e o loop quebrado, retornando os dados agrupados e os centroides gerados em cada grupo.

A implementao do k-means foi feita de forma muito semelhante ideia principal, j que o prprio pandas fornece funes para realizar as aes no banco previstas pelo algoritmo.

Ele comea tomando k linhas aleatrias do banco por meio do sample, que sero os primeiros centroides. Ele ento cria uma coluna centroid no banco e preenche o centroide mais prximo para cada linha. O clculo do centroide mais prximo feito pela funo nearest_centroid, que, dada uma linha do banco e os centroides, seleciona o ndice do centroide cuja distncia euclidiana linha a menor por meio da funo min.

Com o agrupamento inicial feito, o algoritmo define a flag points_change, que define o fluxo do loop, para True e entra em loop at que ela seja falsa, ou seja, nenhum ponto mudou. Ele ento recalcula os centroides, muda points_changed para False e preenche novamente a coluna centroid com os centroides mais prximos de cada linha. Se o valor tiver mudado em relao ao anterior, mais uma iterao deve ser feita, ento a flag mudada para True novamente. Se aps uma iterao nenhum ponto mudar, a flag estar com valor falso, o loop quebrado e o conjunto agrupado e os centroides so retornados.

O reclculo dos centroides feito na funo recalculate_centroids. Como os centroides so simplesmente as mdias de cada grupo, ela simplesmente faz um groupby na base pela coluna de centroides e tira a mdia de cada grupo, gerando os novos centroides.

Uma diferena notvel entre este algoritmo e o k-NN que ele no espera que os dados venham com a coluna de classes. Conforme discutido, o contexto de uso do algoritmo seria justamente o de descobrir uma coluna que divida os dados em classes, ento se presume que ela no conhecida.

def nearest_centroid(row, centroids):
    x = row[row.index != 'centroid']
    idx, min_centroid = min(centroids.iterrows(), key=lambda pair: np.linalg.norm(pair[1]-x))
    return idx

def recalculate_centroids(grouped_data):
    return grouped_data.groupby('centroid').mean()

def kMeans(k, data):
    centroids = data.sample(k, ignore_index = True)
    centroids.index.name = 'centroid'
    
    grouped_data = data.copy()
    grouped_data['centroid'] = [nearest_centroid(row, centroids) for _, row in grouped_data.iterrows()]

    points_changed = True

    while points_changed:
        centroids = recalculate_centroids(grouped_data)
        points_changed = False
        for idx,row in grouped_data.iterrows():
            nearest = nearest_centroid(row, centroids)
            if nearest != row['centroid']:
                grouped_data.loc[idx,'centroid'] = nearest
                points_changed = True
    
    return grouped_data, centroids
