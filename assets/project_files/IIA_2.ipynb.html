Introdução¶

O aprendizado de máquina é a principal técnica de inteligência artificial utilizada atualmente, e um dos seus principais usos é em ciência de dados. Neste contexto, este trabalho visa implementar e analisar o desempenho de dois algoritmos relativamente simples de aprendizado: o k-Nearest Nighbors, um algoritmo de classificação baseado em aprendizado supervisionado, e o k-Means, um algoritmo de agrupamento baseado em aprendizado não supervisionado.

Ambos os algoritmos serão usados para a análise da base de dados Iris, um dataset clássico utilizado no estudo de algoritmos de inteligência artificial. O banco é constituído por informações sobre as três espécies de flores do gênero Iris: Iris setosa, Iris virginica e Iris versicolor. O dataset contém 150 instâncias multivariadas, sendo 50 de cada espécie de Iris, e cada uma contém os dados de comprimento e largura das pétalas e sépalas das flores da instância.

Desse modo, para o primeiro algoritmo, os dados do Iris foram organizados em duas tabelas: iris_train e iris_test. iris_train será usada para o treinamento do algoritmo, sendo composta por 120 linhas, e as demais 30 linhas serão usadas como teste do algoritmo, compondo iris_test. Ambas as tabelas contêm as colunas sepal_length, sepal_width, petal_length e petal_width, contendo as métricas de cada flor, e species, contendo a espécie à qual a flor pertence.

Obs: A execução correta dos códigos requer que os arquivos do dataset Iris fornecidos estejam presentes no ambiente do notebook com os nomes "iris_aleatorio.csv", "iris_treino.csv" e "iris_teste.csv".

Parte 1: k-Nearest Neighbors¶

O k-Nearest Neighbors, ou k-NN, é um algoritmo de classificação utilizado para prever uma determinada característica discreta de uma instância de dados à partir da generalização de um conjunto de treino semelhante fornecido. No caso da base Iris, o objetivo do k-NN é prever, a partir das dimensôes das pétalas e das sépalas, a espécie de uma instância qualquer, à partir das informações da base usadas para o treino do algoritmo.

O k-Nearest Neighbors é, desse modo, um tipo de algoritmo baseado em aprendizado supervisionado. O resultado esperado do algoritmo, a espécie da planta, já é conhecido nos dados, e as classificações do conjunto de treino são usadas para aperfeiçoar o resultado da classificação da instância.

O princípio do k-Nearest Neighbors é bem simples. Os dados contínuos da base, dos quais a classe será predita, podem ser vistos como pontos no espaço. No caso da base Iris, as informações das pétalas e das sépalas formariam um conjunto de pontos em um espaço tetradimensional. Desse modo, dada uma base de treino, uma instância e um determinado valor de k, o k-Nearest Neighbors seleciona os k pontos da base mais próximos da instância, a partir de alguma métrica de distância, e atribui à instância analisada a classe que ocorre mais vezes entre os k. No caso da base Iris, ele escolheria, dentre as três, a espécie a que pertencem a maioria dos elementos entre os k escolhidos e atribuiria à flor que se quer classificar.

O código a seguir é uma implementação do k-NN. Ele utiliza como métrica de distância a distância euclidiana, definida na função euclidean. Ela simplesmente toma duas linhas da tabela, filtra apenas as colunas de dados, sobre as quais a distância será calculada, e calcula a distância entre os pontos fazendo a norma vetorial da diferença dos pontos.

O k-NN em si está implementado na função kNN_classify, e recebe como entrada o número de vizinhos k, que deve ser um inteiro, a base de treino data, que deve ser uma tabela na qual uma das colunas é a categoria de classificação e as demais os demais dados, e a instância instance a ser testada, que deve seguir o mesmo modelo da tabela data (exceto pela coluna de categoria). Opcionalmente, o nome da coluna na qual a categoria a ser classificada pode ser fornecida no parâmetro category_col. Por padrão, se ela não for fornecida, o algoritmo seleciona a última coluna da tabela data

def euclidean(x, y, data_cols):
    return np.linalg.norm(x[data_cols]-y[data_cols])

def kNN_classify(k, data, instance, category_col = None):
    if category_col is None:
        category_col = data.columns[-1]
    
    data_cols = [col for col in data.columns if col != category_col]
    
    k_nearest = data.head(k).copy()
    k_nearest['dist'] = [euclidean(row, instance, data_cols) for _,row in k_nearest.iterrows()]
    
    idx_max = k_nearest['dist'].idxmax()
    max_distance = k_nearest['dist'][idx_max]
    
    for _, row in data.iterrows():
        distance = euclidean(row, instance, data_cols)

        if distance >= max_distance:
            continue
          
        row['dist'] = distance
        k_nearest.loc[idx_max] = row

        idx_max = k_nearest['dist'].idxmax()
        max_distance = k_nearest['dist'][idx_max]
    
    return k_nearest[category_col].mode()[0]

O código em si funciona de forma muito simples e intuitiva. Ele primeiro escolhe k linhas quaisquer da base para ser os k-vizinhos da instância, e para cada uma calcula a distância euclidiana dela à instância dada. Dentre elas, é selecionado o índice da linha mais distante entre as k, em conjunto com o valor de sua distância.

O algoritmo então itera por cada linha da base, calculando sua distância euclidiana da instância e comparando com a distância do vizinho mais distante entre os k. Se ela estiver mais próxima da instância que ele, a linha substitui o vizinho mais distante no conjunto dos k vizinhos e o novo vizinho mais distante é recalculado. Após passar por todas as linhas, o algoritmo retorna a moda da coluna de categorias dos vizinhos, ou seja, a categoria que ocorre com mais frequência entre os k vizinhos mais próximos.

Observe que o algoritmo sempre encontra os k-vizinhos mais próximos. Isso ocorre por dois motivos. Primeiro, um ponto entra no conjunto dos k se, e somente se ele estiver mais próximo que o mais distante entre eles, então um ponto mais distante testado nunca é adicionado no conjunto e um ponto mais próximo testado nunca é ignorado do conjunto. Segundo, todos os pontos da base são testados. Assim, apenas pontos mais próximos são adicionados entre os k, resultando nos k-vizinhos mais próximos.

Um ponto muito importante do k-NN é o fato de seu funcionamento depender da escolha do k adequado para a base utilizada. Assim, tendo implementado o algoritmo, precisamos testar sua eficácia, levando em consideração principalmente a escolha de diferentes valores de k. Para isso, usaremos a divisão do banco mencionada anteriormente. 120 linhas serão utilizadas como treino e as demais 30 como teste.

Uma forma de observar o desempenho do algoritmo é por meio de uma matriz de confusão. Nela, as linhas e colunas contêm as classes da classificação, sendo as linhas o valor real e as colunas o valor predito pelo algoritmo. Assim, para cada instância do conjunto de teste, o k-NN é executado e seu resultado, em conjunto com a classe real já existente na base, determinam a que posição da matriz de confusão a instância pertencerá. A seguinte função foi utilizada para Calcular a matriz de confusão dados o k, a base de treino e a base de teste, sendo a coluna de categorias opcional, como no anterior:

def kNN_test_data(k, train_data, test_data, category_col = None):
    if category_col is None:
        category_col = train_data.columns[-1]
    
    categories = sorted(train_data[category_col].unique())

    confusion = pd.DataFrame(0, index = categories, columns = categories)
    
    for _, row in test_data.iterrows():
        predicted = kNN_classify(k, train_data, row, category_col)
        real = row[category_col]
        confusion.loc[real,predicted] += 1

    confusion.index = pd.MultiIndex.from_product([['Real'],categories])
    confusion.columns = pd.MultiIndex.from_product([['Previsto'],categories], names = ['','k = {}'.format(k)])

    return confusion

As matrizes acima trazem resultados muito interessantes para cada k. O mais evidente é a matriz de confusão para k = 19, que não apresentou erro nenhum, indicando uma classificação perfeita.

Embora a tabela mostre os erros e acertos do algoritmo para cada classe, a interpretação desses dados é de certa forma abstrata e dependente do observador. Precisamos, assim, de métricas que tragam interpretações mais concretas e precisas sobre as informações mostradas na matriz de confusão.

Por essa razão, para cada matriz, foram calculados os valores de acurácia, precisão, revocação e F1:

  • A acurácia é uma métrica geral da matriz, e mede a porcentagem de instâncias identificadas corretamente, ou seja, do total, quantas se encontram na diagonal da matriz.

  • A precisão é uma métrica aplicada a cada classe, e mede a porcentagem de instâncias previstas corretamente dentre todas as previsões da classe. Assim, uma precisão baixa indica que muitas instâncias fora da classe real foram preditas como pertencentes àquela classe, ou seja, um alto número de falsos positivos.

  • A revocação é uma métrica aplicada a cada classe, e mede a porcentagem de instâncias previstas corretamente dentre todas as instâncias reais da classe. Assim, uma revocação baixa indica que muitas instâncias da classe real não foram preditas corretamente, ou seja, um alto número de falsos negativos.

  • O F1 é uma forma de observar a precisão e a revocação de cada classe simultaneamente, fazendo uma média harmônica entre as duas. Uma das principais características da média harmônica é que se um dos elementos for nulo, a média é nula também. Assim, para que uma classe tenha um bom F1 ela precisa ter tanto uma precisão quanto uma revocação boa.

O cálculo das quatro métricas é feito nas seguintes funções:

def accuracy(confusion):
    matrix = confusion.to_numpy()
    acc = matrix.trace()/matrix.sum()
    return pd.DataFrame(acc, index=['Acurácia'], columns = confusion.columns.droplevel(0))

def precision(confusion):
    diags = pd.DataFrame([np.diag(confusion)], index=['Precisão'], columns=confusion.columns.droplevel(0))
    predicts = confusion.sum().to_frame().transpose()
    
    predicts.columns = predicts.columns.droplevel(0)
    predicts.index = ['Precisão']
    return diags/predicts

def recall(confusion):
    diags = pd.DataFrame([np.diag(confusion)], index=['Revocação'], columns=confusion.columns.droplevel(0))
    reals = confusion.sum(axis=1).to_frame().transpose()
    
    reals.columns = reals.columns.droplevel(0)
    reals.index = ['Revocação']
    return diags/reals

def f1(confusion):
    p = precision(confusion)
    p.index = [0]
    r = recall(confusion)
    r.index = [0]

    f_1 = 2*p*r/(p+r)
    f_1.index = ['F1']
    return f_1

Parte 2: k-Means¶

Além de realizar classificações, algoritmos de aprendizado podem também obter novas informações ainda não conhecidas sobre os dados. Este é o princípio do aprendizado não-supervisionado, que, ao contrário do aprendizado supervisionado, não trabalha utilizando resultados esperados para o conjunto de treino, já que estes resultados geralmente ainda não são conhecidos. Assim, algoritmos de aprendizado não-supervisionado geralmente analisam todo o conjunto de dados em busca de associações entre as instâncias.

Dentre estes algoritmos estão os algoritmos de agrupamento, que buscam agrupar os dados em conglomerados que compartilhem entre si informações semelhantes. No caso do dataset Iris, que estamos analisando, uma aplicação imediata do método seria tentar utilizá-lo para classificar as flores em grupos que correspondam às espécies, por exemplo.

O algoritmo de agrupamento que implementaremos será o k-Means. Seu funcionamento também é relativamente simples, e seu objetivo é, dada uma base de dados e um número k de grupos, dividí-la em k grupos de modo que pontos em um mesmo grupo estejam mais próximos entre si em relação aos demais.

Para isso, o k-Means começa selecionando k pontos aleatórios da base para cerem os centroides de cada grupo, e associando cada ponto da base ao grupo do centroide mais próximo a ele. O algoritmo então entra em um loop, no qual ele recalcula os centroides, fazendo a média dos pontos de cada grupo (vide o nome, k-Means), e reagrupando os pontos de acordo com o centroide mais próximo. Quando todos os pontos continuarem no mesmo grupo após uma iteração do loop, os grupos convergiram e o loop é quebrado, retornando os dados agrupados e os centroides gerados em cada grupo.

A implementação do k-means foi feita de forma muito semelhante à ideia principal, já que o próprio pandas fornece funções para realizar as ações no banco previstas pelo algoritmo.

Ele começa tomando k linhas aleatórias do banco por meio do sample, que serão os primeiros centroides. Ele então cria uma coluna centroid no banco e preenche o centroide mais próximo para cada linha. O cálculo do centroide mais próximo é feito pela função nearest_centroid, que, dada uma linha do banco e os centroides, seleciona o índice do centroide cuja distância euclidiana à linha é a menor por meio da função min.

Com o agrupamento inicial feito, o algoritmo define a flag points_change, que define o fluxo do loop, para True e entra em loop até que ela seja falsa, ou seja, nenhum ponto mudou. Ele então recalcula os centroides, muda points_changed para False e preenche novamente a coluna centroid com os centroides mais próximos de cada linha. Se o valor tiver mudado em relação ao anterior, mais uma iteração deve ser feita, então a flag é mudada para True novamente. Se após uma iteração nenhum ponto mudar, a flag estará com valor falso, o loop é quebrado e o conjunto agrupado e os centroides são retornados.

O recálculo dos centroides é feito na função recalculate_centroids. Como os centroides são simplesmente as médias de cada grupo, ela simplesmente faz um groupby na base pela coluna de centroides e tira a média de cada grupo, gerando os novos centroides.

Uma diferença notável entre este algoritmo e o k-NN é que ele não espera que os dados venham com a coluna de classes. Conforme discutido, o contexto de uso do algoritmo seria justamente o de descobrir uma coluna que divida os dados em classes, então se presume que ela não é conhecida.

def nearest_centroid(row, centroids):
    x = row[row.index != 'centroid']
    idx, min_centroid = min(centroids.iterrows(), key=lambda pair: np.linalg.norm(pair[1]-x))
    return idx

def recalculate_centroids(grouped_data):
    return grouped_data.groupby('centroid').mean()

def kMeans(k, data):
    centroids = data.sample(k, ignore_index = True)
    centroids.index.name = 'centroid'
    
    grouped_data = data.copy()
    grouped_data['centroid'] = [nearest_centroid(row, centroids) for _, row in grouped_data.iterrows()]

    points_changed = True

    while points_changed:
        centroids = recalculate_centroids(grouped_data)
        points_changed = False
        for idx,row in grouped_data.iterrows():
            nearest = nearest_centroid(row, centroids)
            if nearest != row['centroid']:
                grouped_data.loc[idx,'centroid'] = nearest
                points_changed = True
    
    return grouped_data, centroids
